# professional-template-standard

## Цель
Гарантировать, что все проекты на базе `next.js-starter` соответствуют высоким стандартам дизайна и надежности. Это включает читаемую типографику и автоматическую проверку работоспособности критических серверных функций.

## Правила UI/UX

### 1. Выравнивание текста
- **ЗАПРЕЩЕНО**: Использовать `text-justify` (выравнивание по ширине). Это создает неравномерные пробелы ("реки"), которые затрудняют чтение.
- **ОБЯЗАТЕЛЬНО**: Использовать выравнивание по левому краю (`text-left`).
- **ИСКЛЮЧЕНИЕ**: Центрирование (`text-center`) допускается только для коротких заголовков или декоративных элементов.

### 2. Длина строки и ширина блоков
- **ПРАВИЛО**: Текстовые блоки длиннее 2 строк не должны растягиваться на всю ширину экрана.
- **ДЕЙСТВИЕ**: Ограничивать ширину контейнеров параграфов классами `max-w-2xl` (~650px) или `max-w-3xl`.
- **ОБОСНОВАНИЕ**: Оптимальная длина строки (45-75 символов) значительно снижает когнитивную нагрузку при чтении.

### 3. Вертикальный ритм
- **ПРАВИЛО**: Использовать `leading-relaxed` для основного контента.
- **ПРАВИЛО**: Соблюдать единообразные отступы между блоками (`space-y-4`, `mb-6`).

## Правила надежности (Health)

### 1. Проверка Кронов
- **ПРАВИЛО**: Любой проект с крон-задачами должен включать таблицу логирования `cron_logs` и скрипт `scripts/check-cron-health.mjs`.
- **ОБЯЗАТЕЛЬНО**: Проверка здоровья кронов должна быть частью процесса `/pre-deploy`.

## Процедура проверки
1. Поиск `text-justify` по коду и его удаление.
2. Проверка `<p>` тегов на наличие ограничений `max-width`.
3. Запуск `node scripts/check-cron-health.mjs` для подтверждения статуса фоновых задач.
